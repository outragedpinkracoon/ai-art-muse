# Troubleshooting

Common issues and solutions for the AI Image CLI.

## Installation Issues

### "mflux-generate-flux2: command not found"

Python's bin directory isn't on your PATH.

```bash
# Add to your shell profile (~/.zshrc or ~/.bashrc)
export PATH="$HOME/Library/Python/3.9/bin:$PATH"

# Or find where it was installed
which mflux-generate-flux2
```

### "pip install mflux" fails

```bash
# Try upgrading pip first
pip install --upgrade pip

# Install with --user flag if permissions issue
pip install --user mflux

# If on Apple Silicon, ensure you have the right Python
brew install python3
pip3 install mflux
```

### "HF_TOKEN" error

The model download requires a Hugging Face token.

```bash
# Get your token from https://huggingface.co/settings/tokens
export HF_TOKEN="your_token_here"

# Make it permanent
echo 'export HF_TOKEN="your_token_here"' >> ~/.zshrc
```

### Ollama model won't pull

```bash
# Check if ollama is running
ollama list

# If not, start it
ollama serve

# Try pulling again
ollama pull qwen3-vl:8b
```

## Runtime Issues

### Out of memory

Generation peaks around ~17GB. On a 16GB Mac this is tight. If you're hitting memory limits:

```bash
# Reduce steps
muse generate "prompt" --steps 8

# Disable LoRA (reduces memory)
muse generate "prompt" --no-lora

# Close other applications (browsers, etc.)
```

### Generation is slow

First generation loads the model (~16GB) and is slow. Subsequent runs are faster.

Rough timings below were measured on an M2 Pro (32GB) — yours will vary by chip and memory:

| Setting | Approx time |
|---------|------|
| 12 steps, with LoRA (txt2img) | ~42s |
| 8 steps (edit) | ~90s |
| 8 steps, no LoRA | ~30s |

### "Model not found" error

The model may not have downloaded completely.

```bash
# Clear cache and retry
rm -rf ~/.cache/huggingface/hub/models--black-forest-labs--FLUX.2-klein-4B
export HF_TOKEN="your_token_here"
muse generate "test"  # triggers redownload
```

### "Steps must be 2 or more"

mflux divides internally by `steps - 1`, so a single step is a hard error. Use `--steps 2` or higher.

### Image generation looks wrong

Klein 4B is distilled and responds differently than full Flux.

**Problem:** Style prose not taking effect.

**Solution:** Style comes from both prose AND LoRA. Make sure you're using both:

```bash
# Prose + LoRA — full style preset, strongest effect
muse generate "prompt" --style chalk

# LoRA only, no prose — subtle; the LoRA alone is a light touch
muse generate "prompt" --bare

# Neither — closest to unstyled / photoreal
muse generate "prompt" --bare --no-lora
```

**Problem:** Colors are oversaturated or distorted.

**Solution:** Lower the LoRA scale.

```bash
muse generate "prompt" --lora-scale 0.6
```

**Problem:** Subject not matching prompt.

**Solution:** Increase steps.

```bash
muse generate "prompt" --steps 20
```

### Edit not working

**Problem:** Edit instruction changes too much.

**Solution:** Keep edits terse and specific. "make the hat blue" works better than "change the colour of the hat to a nice shade of blue".

**Problem:** Edit changes the whole image, or you need a new object/scene rather than a local tweak.

**Solution:** The model anchors on the source layout, so it's weak at adding large new objects or new compositions. Use `regen` instead:

```bash
muse regen output.png "new subject"
```

### Regen/Restyle can't read source image

These commands require the source image to have mflux's metadata blob embedded in its PNG EXIF.

**Solution:** Only use `regen` and `restyle` on images generated by `muse generate`. The metadata is saved automatically — no flag needed. If the image came from elsewhere (downloaded, screenshotted, edited externally), use `--edit` instead.

### "Unknown style(s)" error

You passed a `--style` name that isn't in `prompts/styles.json`. The error lists the available presets — current ones are `illustration`, `sketch`, `chalk`, `delicate`, `character`, `object`. Layers are comma-separated, e.g. `--style chalk,character`.

### "--style is ignored in edit mode" warning

Prose style never applies during `--edit` — appending it would corrupt the edit instruction. Use `--lora-style` to change the visual look during an edit instead.

## Model Issues

### Vision model not responding

```bash
# Check if it's running
ollama ps

# If not, it should load automatically on first critique
# If it hangs, try pulling manually
ollama pull qwen3-vl:8b
```

### Imagine model too slow

The imagine model is large (12B) and loads on first question. It's stopped automatically when the loop ends, so it reloads every fresh `muse imagine` call. To use a smaller model instead, edit `IMAGINE_MODEL` in `lib/config.rb`.

### "No such file or directory" for the imagine model

```bash
# Pull the model
ollama pull gemma4:12b-mlx
```

## File and Path Issues

### "output" directory missing

The CLI creates it automatically on first `muse generate`. If something's gone wrong:

```bash
mkdir -p output
```

### Permission denied on muse

```bash
chmod +x muse
./muse generate "prompt"
```

### Can't find the muse command

Add the repo to your PATH or symlink:

```bash
# Add to PATH
export PATH="/path/to/ai-image:$PATH"

# Or symlink
ln -s /path/to/ai-image/muse ~/bin/muse
```

## Performance Tuning

### Faster generation

```bash
# Use fewer steps
muse generate "prompt" --steps 8

# Disable LoRA
muse generate "prompt" --no-lora
```

### Reduce memory usage

```bash
# Close other applications
# Use --no-lora
```

### Improve quality

```bash
# More steps (diminishing returns after 16)
muse generate "prompt" --steps 20

# Pin a good seed for iteration
muse generate "prompt" --seed 42

# Use regen to iterate on a good base
muse regen output/good.png "new subject"
```

## Still having issues?

1. Run with `--verbose` to see the full mflux command being executed
2. Check `smoke-test/smoke_error.txt` for detailed error logs from the last smoke test run
3. Run the smoke test to isolate which component is failing:
   ```bash
   ./smoke-test/smoke           # run every step
   ./smoke-test/smoke chat      # run just one step (also: generate edit single compare ask imagine)
   ```

### Getting help

- Check the [Install Guide](install-guide.md) for dependency details
- Verify all models are present: `ollama list`
- Ensure `mflux` works standalone: `mflux-generate-flux2 --help`
